Color()
Component implementing various color palettes and coloring policies. When this component is available for a widget, its API is exposed via the .color namespace.
Color.deficiency(type)
Emulates a color vision deficiency. Useful for testing the widget for accessibility. The deficiency emulation is implemented using the algorithm here .

Parameters

NameTypeDescription
typestringType of deficiency to emulate. Supported values: achromatomaly, achromatopsia, deuteranomaly, deuteranopia, protanomaly, protanopia, tritanomaly, tritanopia. If type is not specified, it restores the true colors.

Color.palette([palette[, on]])
Sets the color palette. Supported palettes:
  • Default color palette: A variant of the 8 color colorblind friendly qualitative palette by Bang Wang. See this paper and this post for details. This color palette is also available by passing palette-wong to the method.
    Colors:
  • Built-in color palette: One of the built-in palettes that support the vast majority of color vision deficiencies:
    • palette-lightThe 9 color colorblind friendly qualitative palette designed by Paul Tol. See this post for details.
      Colors:
  • Single color: The specified color is used for all plots.
  • Custom palette: A custom color palette with arbitrary association with the plots. In this case, the colors are assigned to the plots in the order they are added/built.
  • Custom color mapping: Each plot has the color specified as the value for the property with the same name as the plot's key.

Parameters

NameTypeDescription
palettestring, ObjectoptionalColor palette to set. If not specified, the default policy is set. If string, it's either a built-in palette or a the color represented by the string is used for all plots. If an array of strings, it's colors are assigned to plots as they are added to the widget without any specific association between colors and plots. If an object, it is used as a mapping from the plot names to the colors.
onoptionalTransformation that is used before mapping the plot to a color. The plot name is passed to this method as parameters before determining the color. Default value is d => d.
